# 사전 안내와 준비 기준

워크숍을 시작하기 전에 참여자가 무엇을 준비해야 하는지 짧게 안내합니다. 이 문서는 긴 과제 안내가 아니라, 현장에서 질문을 덜 하고 3시간 MVP로 빨리 들어가기 위한 준비 기준입니다.

## 참가자에게 미리 안내할 내용

아래 네 가지를 한 문장씩 생각해 오게 합니다.

- 현장 문제: 지금 반복해서 처리하느라 불편한 업무는 무엇인가요?
- 반복 업무: 사람이 읽고, 옮기고, 판단하고, 다시 쓰는 지점은 어디인가요?
- 자료 상태: 실제 자료가 있나요, 아니면 가상 샘플로 시작해야 하나요?
- 기대 결과: 발표장에서 어떤 형태의 결과물을 보면 도움이 되나요?

단, 워크숍 초반에 `샘플 입력`과 `기대 산출물`을 바로 확정하라고 요구하지 않습니다. 사회혁신가가 답하기 어려워하면 업무 문제를 먼저 듣고, 스킬이 자료와 결과물 후보를 제안합니다.

## 샘플 자료 준비 기준

가능하면 행사 전에 아래 중 하나를 준비합니다.

- 개인정보를 제거한 실제 문서, 표, 신청서, 상담 기록, 문의, 매뉴얼, 화면 캡처
- 원본 구조를 닮은 더미 데이터
- 공개 가능한 예시 자료
- 자료가 전혀 없을 때 만들 수 있는 가상 샘플 설명

샘플은 작아도 됩니다. 3시간 MVP에서는 전체 데이터가 아니라 입력 1개로 업무 흐름을 보여주는 것이 우선입니다.

## 개인정보와 내부자료 처리

아래 정보는 원본 그대로 넣지 않습니다.

- 이름, 연락처, 주소, 주민등록번호, 계좌번호, 이메일 등 직접 식별자
- 상담 원문, 내부 회의록, 민감한 사정, 접근권한 세부 정보
- 비밀번호, 토큰, API key, secret, 계정 정보
- 공개 동의가 없는 원본 파일이나 화면

처리 방식은 `익명화`, `마스킹`, `삭제`, `더미 생성`, `화면 시연만` 중 하나로 기록합니다. 공개 사례에 쓸 수 없으면 `내부 공유만` 또는 `공개 불가`로 표시합니다.

## 최종 제출/아카이빙 기준

워크숍 결과는 구현 결과물만으로 끝나지 않습니다. 아래 파일을 함께 남깁니다.

- `PLAN.md`: 문제정의와 3시간 실행 계획
- `workshop.json`: 질문 답변과 구현 결과를 표준 필드로 정리한 기준 데이터
- `MEMORY.md`: 구현 중 결정과 변경 기록
- `WORKFLOW_ANALYSIS.md`: AI Agent Workflow 라이브러리용 구조화 분석
- `CASE_STUDY.md`: 공개 가능한 익명화 사례 요약

`workshop.json`은 제출과 아카이빙의 기준 파일입니다. 마크다운 산출물을 다시 만들거나, 여러 팀 결과를 모아 분석할 때 이 파일을 사용합니다.
