# 결과물 형태 선택 기준

업무 문제를 들은 뒤 가능한 결과물 후보를 제안할 때 사용합니다. 결과물 형태는 문제 유형과 다릅니다. 문제 유형은 질문을 줄이기 위한 분류이고, 결과물 형태는 발표장에서 실제로 보여줄 구현물입니다.

## 공통 원칙

- 결과물 후보는 2-4개만 제안합니다.
- 모든 후보는 3시간 안에 최소 데모가 가능해야 합니다.
- 명백한 정답 하나와 들러리 선택지를 만들지 않습니다.
- 각 후보 설명에는 입력, 출력, 사람 검토 지점을 한 줄로 적습니다.
- 로그인, 결제, 권한, 자동 발송, 외부 API 연동, 실시간 DB 저장은 기본적으로 다음 단계로 둡니다.

## 결과물 형태

### 1. 자동화 도구

적합한 상황:

- 반복 문서, 표, 문의, 신청서, 상담 기록을 처리합니다.
- 사용자는 결과 파일이나 표만 받아도 업무가 줄어듭니다.

3시간 MVP 예시:

- CSV 업로드 -> 위험/누락/우선순위 표
- 문서 텍스트 입력 -> 요약과 확인 질문
- 문의 3개 입력 -> 근거가 붙은 답변 초안

사람 검토 지점:

- 분류 기준, 누락 항목, 최종 발송/확정 전 검토

### 2. 웹앱

적합한 상황:

- 사용자가 입력하고 바로 결과를 확인해야 합니다.
- 화면이 있어야 발표장에서 흐름이 이해됩니다.

3시간 MVP 예시:

- 신청서 붙여넣기 화면 -> 검토 결과 화면
- 캠페인명 입력 화면 -> 홍보 문구 생성 화면
- 대상자 정보 입력 화면 -> 안내 초안 결과 화면

사람 검토 지점:

- 결과 화면에서 수정, 확정, 보류 중 하나를 고르는 지점

### 3. 온보딩 페이지

적합한 상황:

- 외부 참여자, 봉사자, 활동가, 수혜자가 절차를 따라야 합니다.
- 자료 안내와 행동 유도가 중요합니다.

3시간 MVP 예시:

- 프로그램 참여 안내 -> 개인별 준비물 체크리스트
- 봉사자 역할 설명 -> 상황별 대응 가이드
- 신청 절차 설명 -> 누락 없이 제출하는 안내 페이지

사람 검토 지점:

- 안내 문구의 정확성, 민감한 표현, 조직 정책 반영 여부

### 4. admin 화면

적합한 상황:

- 담당자가 여러 항목을 검토, 승인, 배정, 라우팅해야 합니다.
- 최종 판단은 사람이 해야 합니다.

3시간 MVP 예시:

- 신청 3건 -> 검토 플래그와 담당자 메모 화면
- 제보 목록 -> 부서 배정 후보와 근거 화면
- 후원자 목록 -> 연락 우선순위와 확인 필요 표시

사람 검토 지점:

- 승인/보류/배정/최종 판단 버튼 또는 체크리스트

### 5. 대시보드

적합한 상황:

- 반복적으로 지표를 확인하거나 비교해야 합니다.
- 표나 그래프가 의사결정을 돕습니다.

3시간 MVP 예시:

- 설문 결과 -> 핵심 지표 3개와 코멘트 요약
- 후원 데이터 -> 유지/이탈 위험 요약
- 신청자 목록 -> 카테고리별 현황표

사람 검토 지점:

- 계산 기준, 필터 기준, 해석 문구 확인

### 6. 문서/메시지 초안

적합한 상황:

- 최종 산출물이 메일, 안내문, 보고서, 회의록, 문자, 제안서 초안입니다.
- 자동 발송보다 초안 품질과 검수가 중요합니다.

3시간 MVP 예시:

- 회의록 -> 실행 항목과 담당자 표
- 캠페인 설명 -> FAQ와 홍보 문구
- 상담 기록 -> 후속 안내 문자 초안

사람 검토 지점:

- 최종 표현, 사실관계, 발송 가능 여부
