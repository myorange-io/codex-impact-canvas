# 3시간 완결형 MVP 범위 판단

3시간 MVP는 작은 장난감 기능이 아니라, 좁게 자른 실제 문제를 시작부터 검토 가능한 결과까지 끝내는 세로 조각입니다. 조직 전체 문제를 완전히 해결하려고 하지 말고, 사회혁신가가 "이 한 업무는 오늘 당장 덜 힘들겠다"고 느낄 만큼 한 업무 단위를 끝까지 해결합니다.

## 기본 원칙

- 목표는 `얕은 기능 1개`가 아니라 `좁은 문제 1개를 끝까지 처리하는 완결형 업무 흐름`입니다.
- 입력은 하나의 업무 묶음으로 제한합니다. 묶음 크기는 이 프로젝트의 실제 자료 중 발표장에서 다룰 수 있는 최소 단위로 정합니다.
- 출력은 의사결정, 전달, 검토, 기록 중 하나에 바로 쓸 수 있는 완성 산출물이어야 합니다.
- 중간 단계는 2-4개까지 허용합니다. 단계명은 이 프로젝트의 실제 처리 흐름에서 가져와 작성합니다.
- 최종 판단, 외부 발송, 결제, 실제 접수, 권한 변경, 운영 DB 저장은 사람이 하거나 다음 단계로 둡니다.
- 사용자가 반복 제작물을 줄이고 싶어 하면 MVP는 검수표가 아니라 반복 제작물 자동 생성이어야 합니다. 검수표와 체크리스트는 자동 생성 결과의 안전장치로 붙입니다.

## 하드 게이트

아래 중 하나라도 실패하면 3시간 MVP 밖이거나 축소가 필요합니다.

- 실제 샘플 입력 또는 익명화 샘플이 없다.
- 발표장에서 볼 수 있는 최종 산출물이 없다.
- 누가 결과를 검토하고 어디에 쓰는지 정해지지 않았다.
- 성공/실패를 샘플 결과로 판단할 수 없다.
- 외부 API, 로그인/권한, 배포, 결제, 자동 발송, 운영 DB 저장이 필수다.
- 개인정보나 민감정보를 안전하게 더미/익명화/화면 시연으로 바꿀 수 없다.

## 판정 공식

먼저 문제 매력도를 확인하고, 그 다음 3시간 적합도를 판단합니다.

```text
문제 매력도 = 중요도 x 미충족도 x 반복성
3시간 적합도 = 현장효용 + 완결성 + 데이터준비도 + 구현쉬움 + 검수가능성 - 위험패널티
```

점수는 각 항목 1-5점입니다. 위험패널티만 0-5점입니다.

- 18점 이상: 3시간 MVP 안. 지금 범위로 진행합니다.
- 14-17점: 축소 필요. 입력 묶음, 업무 단계, 외부 연동, 자동화를 하나 줄입니다.
- 13점 이하: 3시간 MVP 밖. 더 좁은 문제로 다시 자릅니다.

단, 구현쉬움 또는 데이터준비도가 2점 이하이면 총점과 무관하게 `축소 필요`로 봅니다. 하드 게이트가 실패하면 총점과 무관하게 `밖`입니다.

## 단순해 보일 때 깊이를 더하는 순서

사회혁신가가 보기에는 단순한 초안 생성만으로 충분하지 않을 수 있습니다. 그럴 때는 새 시스템을 붙이지 말고 아래 순서로 업무 완결성을 높입니다.

1. 결과에 근거 문장이나 계산 근거를 붙입니다.
2. 예외/애매함/확인 필요 항목을 분리합니다.
3. 담당자가 바로 쓸 검토 체크리스트를 붙입니다.
4. 이 프로젝트에서 실제 다음 행동에 쓰는 산출물을 붙입니다.
5. 입력 1개가 너무 약하면 같은 형식의 최소 입력 묶음으로 보여줍니다.

선택지가 여러 개면 `references/multiple-choice.md`의 객관식 플래닝으로 모두 실행 가능한 완결형 대안을 비교합니다. 추천안은 항상 3시간 적합도와 현장효용을 기준으로 고르며, 명백히 약한 들러리 선택지는 만들지 않습니다.

선택지는 사용자의 자동화 기대 수준을 모두 충족해야 합니다. 사용자가 특정 반복 산출물 자동 생성을 원한다면 객관식은 이 프로젝트에서 실제로 만들 산출물 이름으로 나누고, 단순 검수표나 입력 보조는 별도 선택지가 아니라 보강 요소로 둡니다.

## 밖으로 빼야 할 복잡도

아래는 사회혁신가에게 중요해 보여도 3시간 MVP 본문이 아니라 다음 단계나 빠르게 끝났을 때 후보로 둡니다.

- 운영 서비스 배포
- 로그인, 권한, 관리자 화면
- 실제 DB 저장과 권한별 조회
- 공개 챗봇 또는 대외 자동 상담
- 문자, 메일, 기관 접수 자동 발송
- 전체 파일 일괄 처리
- 최종 선정, 지원 확정, 상담 판정 자동화

## 축소 선택지

3시간 적합도가 `축소 필요` 또는 `밖`이면 아래 중 하나를 객관식으로 고르게 합니다.

- A. 입력 줄이기: 이 프로젝트의 전체 자료 대신 발표 가능한 최소 입력 묶음으로 제한합니다.
- B. 단계 줄이기: 이 프로젝트의 전체 흐름 대신 사람이 검토할 핵심 산출물까지만 만듭니다.
- C. 연동 빼기: 이 프로젝트의 외부 연동은 다음 단계로 두고 수동 입력과 복사용 결과로 시연합니다.
- D. 판단 낮추기: 이 프로젝트의 최종 판단 자동화 대신 근거, 보류 사유, 사람 검토 지점만 만듭니다.
