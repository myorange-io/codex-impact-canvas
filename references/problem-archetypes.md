# 문제 유형별 질문

현장 문제를 들은 뒤 아래 유형 중 하나로 분기합니다. 유형은 라벨이 아니라 질문을 줄이기 위한 도구입니다. 여러 유형이 섞이면 발표장에서 보여줄 핵심 산출물 기준으로 하나만 고릅니다.

## 공통 규칙

- 첫 세 질문 안에 문제 유형, 샘플 입력 업무 묶음, 기대 최종 산출물을 정합니다.
- 여러 유형이 섞이면 `references/multiple-choice.md`의 객관식 프롬프팅으로 2-3개 후보를 제시하고 추천 유형을 하나 고릅니다.
- 제품 PRD, 시장성, 가격 전략, GTM 검토로 확장하지 않습니다.
- 오늘 만들 것은 3시간 안에 시연 가능한 입력 업무 묶음 1개 -> 검토 가능한 최종 산출물 1개 흐름입니다. 중간 단계 2-4개는 허용하지만 같은 문제 해결에 속해야 합니다.
- 자동 연동, 계정 권한, 일괄 처리, 배포, 결제, 문자/메일 자동 발송은 기본적으로 다음 단계로 둡니다.
- 민감정보가 있으면 샘플, 더미, 익명화, 화면 시연 중 하나로 제한합니다.

## 1. 문서/콘텐츠 초안 또는 변환

대표 상황:

- 제안서, 보고서, 감사장, 메일, 문자, 안내문, PPT, 자료집, 회의록을 반복 작성합니다.
- 기존 문서를 읽고 다른 형식의 문서로 옮깁니다.

반드시 묻는 질문:

- "오늘 넣어볼 원본 문서나 텍스트 1개는 무엇인가요?"
- "그 원본에서 만들어낼 최종 문서 1개는 무엇인가요?"
- "사람이 최종 수정할 때 꼭 확인해야 하는 기준은 무엇인가요?"
- "AI가 실제로 대신 만들어야 하는 반복 제작물은 무엇인가요?"

MVP 예시:

- 신청서 텍스트 1개 -> 누락 항목 목록, 근거, 안내 메일 초안, 검토 체크리스트
- 캠페인 자료 1개 -> 참여자 FAQ, 홍보 문구 초안, 위험 표현 검토표
- 회의록 1개 -> 실행 항목, 담당자 표, 공유 메시지 초안

다음 단계로 뺄 것:

- 여러 파일 일괄 처리
- 실제 발송 자동화
- 디자인 완성도 높은 PPT 전체 제작

객관식 후보 예시:

- A. 단일 문서 자동 제작형: 원본 데이터 -> 가장 중요한 문서 1종 초안
- B. 복수 제작물 자동 생성형: 원본 데이터 -> 서로 연결된 제작물 2종 초안
- C. 커뮤니케이션 패키지형: 원본 데이터 -> 제작물 초안과 안내/공유 문구

## 2. 데이터 정리/분석/대시보드

대표 상황:

- 엑셀, CSV, 구글시트, 후원관리 데이터, 신청자 목록, 설문 결과를 반복 정리합니다.
- 계산 기준이 복잡하거나 매번 피벗/필터를 다시 만듭니다.

반드시 묻는 질문:

- "오늘 넣어볼 원본 표 1개는 무엇인가요?"
- "사람이 지금 직접 계산하거나 분류하는 기준 1개는 무엇인가요?"
- "발표장에서 보여줄 핵심 지표나 표 1개는 무엇인가요?"

MVP 예시:

- 후원자 raw data 샘플 -> 3/6/9개월 유지율 요약표, 이탈 후보 표시, 확인 질문
- 신청팀 목록 -> 심사 마스터파일 초안, 누락값 표시, 검토 우선순위
- 캠페인 데이터 -> 핵심 지표 3개가 있는 대시보드 초안, 이상값 메모

다음 단계로 뺄 것:

- 실시간 데이터베이스 연동
- 전체 BI 시스템
- 권한별 대시보드

객관식 후보 예시:

- A. 정리표형: 원본 표 -> 검토 가능한 마스터표
- B. 지표 요약형: 원본 표 -> 핵심 지표와 이상값 메모
- C. 우선순위형: 원본 표 -> 처리 순서와 확인 질문

## 3. 문의응대/매뉴얼/상담 보조

대표 상황:

- 반복 질문, 상담 기록, 행정 매뉴얼, 내부 FAQ를 기반으로 답변을 만듭니다.
- 담당자가 매번 자료를 찾아 문구를 다시 씁니다.

반드시 묻는 질문:

- "오늘 테스트할 실제 질문 예시 3개는 무엇인가요?"
- "답변이 참고해야 할 근거 자료는 무엇인가요?"
- "AI가 틀리면 위험한 답변 유형은 무엇이고, 사람이 어디서 검토해야 하나요?"

MVP 예시:

- 캠페인 질문 3개 -> 근거가 붙은 답변 초안과 답변 위험도 표시
- 행정 매뉴얼 일부 -> 활동가용 절차 안내
- 상담 기록 요약 -> 다음 안내 문자 초안과 사람이 확인할 민감 표현

다음 단계로 뺄 것:

- 공개 챗봇 배포
- 개인별 자동 상담
- 민감 사안 최종 판단 자동화

객관식 후보 예시:

- A. 근거 답변형: 질문 묶음 -> 근거가 붙은 답변 초안
- B. 상담 보조형: 상담 기록 -> 확인 필요 신호와 후속 질문
- C. 매뉴얼 안내형: 매뉴얼 일부 -> 절차 안내와 예외 처리

## 4. 분류/추천/매칭/리서치

대표 상황:

- 뉴스, 리드, 자원, 지원서, 참여자 피드백, 사회공헌 정보 등을 분류하거나 연결합니다.
- 사람이 좋은 결과와 나쁜 결과를 구분하는 기준을 갖고 있습니다.

반드시 묻는 질문:

- "좋은 결과 예시 1개와 나쁜 결과 예시 1개는 무엇인가요?"
- "AI가 따라야 하는 분류, 추천, 매칭 기준은 무엇인가요?"
- "기준에 맞지 않거나 애매한 경우에는 어떻게 처리해야 하나요?"

MVP 예시:

- 뉴스 기사 5개 -> 사업/팀/깊이 기준 분류표와 추천 근거
- 복지자원 목록 -> 대상자 욕구별 후보 3개와 부적합 사유
- 신청서 3개 -> 심사 마스터파일, 검토 플래그, 보류 사유

다음 단계로 뺄 것:

- 전체 웹 크롤링
- 외부 서비스 자동 등록
- 최종 선정 자동화

객관식 후보 예시:

- A. 분류표형: 여러 입력 -> 기준별 분류표와 근거
- B. 추천 후보형: 대상 조건 -> 후보 목록과 부적합 사유
- C. 리서치 정리형: 자료 묶음 -> 요약, 태그, 다음 검토 항목

## 5. 웹앱/시스템 구축 욕구

대표 상황:

- 플랫폼, 시스템, 웹앱, DB, 백엔드, 대시보드, 신청 페이지를 만들고 싶어 합니다.
- 요구가 크지만 3시간 안에 전체 구현은 어렵습니다.

반드시 묻는 질문:

- "오늘 만들 화면 1개만 고르면 어떤 화면인가요?"
- "그 화면에서 사용자가 입력하거나 선택하는 값 1개는 무엇인가요?"
- "오늘은 수동으로 남길 부분과 다음 단계로 넘길 부분은 무엇인가요?"

MVP 예시:

- 캠페인명 입력 화면 -> 문구 초안, 위험 표현 검토, 복사용 결과 화면
- 신청서 업로드 화면 -> 마스터파일 미리보기와 누락 항목 표시
- 회원 정보 입력 화면 -> 맞춤 안내 문자 초안과 발송 전 확인표

다음 단계로 뺄 것:

- 로그인/권한
- 실제 DB 저장
- 결제/발송/외부 API 자동 실행

객관식 후보 예시:

- A. 화면 데모형: 입력 화면 1개 -> 결과 화면 1개
- B. 업무 완결형: 수동 입력 -> 검토표/메시지/기록용 산출물
- C. 운영 준비형: 샘플 데이터 -> 운영 흐름과 예외 처리표

## 6. MVP 미정형

대표 상황:

- "아직 잘 모르겠다", "현실성 있는 것을 먼저", "조언을 받고 싶다", "플랫폼 구축"처럼 기능이 정해지지 않았습니다.
- 문제는 있지만 반복 업무나 산출물이 흐릿합니다.

반드시 묻는 질문:

- "가장 자주 반복되는 업무 1개는 무엇인가요?"
- "그 업무에서 오늘 사람이 덜 해도 되는 행동 1개는 무엇인가요?"
- "그 행동을 줄였다는 것을 보여줄 샘플 입력과 출력은 무엇인가요?"

MVP 예시:

- 반복 메일 답변 3건 -> 답변 초안, 근거, 보류 사유
- 흩어진 자료 1묶음 -> 정리표, 누락 항목, 다음 행동
- 업무 매뉴얼 일부 -> 담당자 체크리스트와 예외 처리 안내

다음 단계로 뺄 것:

- 조직 전체 업무 자동화
- 문제 탐색 컨설팅 전체
- 장기 제품 로드맵

객관식 후보 예시:

- A. 가장 자주 반복되는 업무를 줄입니다.
- B. 가장 위험한 누락을 먼저 발견합니다.
- C. 가장 많이 다시 쓰는 문서나 메시지를 만듭니다.
