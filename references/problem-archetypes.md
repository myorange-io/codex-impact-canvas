# 문제 유형별 질문

현장 문제를 들은 뒤 아래 유형 중 하나로 분기합니다. 유형은 라벨이 아니라 질문을 줄이기 위한 도구입니다. 여러 유형이 섞이면 발표장에서 보여줄 핵심 산출물 기준으로 하나만 고릅니다.

## 공통 규칙

- 업무 문제를 먼저 확인한 뒤 문제 유형을 가늠합니다. 샘플 입력과 기대 출력은 자료 확인과 결과물 선택 단계에서 함께 정합니다.
- 제품 PRD, 시장성, 가격 전략, GTM 검토로 확장하지 않습니다.
- 오늘 만들 것은 3시간 안에 시연 가능한 입력 1개 -> 출력 1개 흐름입니다.
- 3시간 MVP 범위가 애매하면 `references/mvp-scope.md`의 하드 게이트와 판정 공식을 사용합니다.
- 여러 실행안이 섞이면 `references/multiple-choice.md`에 따라 모두 실행 가능한 객관식 후보로 수렴합니다.
- 자동 연동, 계정 권한, 일괄 처리, 배포, 결제, 문자/메일 자동 발송은 기본적으로 다음 단계로 둡니다.
- 민감정보가 있으면 샘플, 더미, 익명화, 화면 시연 중 하나로 제한합니다.
- 자료가 없으면 유형별 가상 샘플을 만들어도 됩니다. 단, 산출물에는 `가상 샘플`이라고 표시합니다.

## 1. 문서/콘텐츠 초안 또는 변환

대표 상황:

- 이 프로젝트에서 반복 작성하는 문서나 메시지가 있습니다.
- 이 프로젝트의 기존 자료를 읽고 다른 형식의 산출물로 옮깁니다.

반드시 묻는 질문:

- "오늘 넣어볼 원본 문서나 텍스트 1개는 무엇인가요?"
- "그 원본에서 만들어낼 최종 문서 1개는 무엇인가요?"
- "사람이 최종 수정할 때 꼭 확인해야 하는 기준은 무엇인가요?"

프로젝트 기반 MVP 예시 구성:

- `[이 프로젝트의 원본 자료 1개] -> [현재 프로젝트에서 필요한 최종 문서/메시지 초안]`
- `[반복 작성하는 자료] -> [사람이 검토할 핵심 항목과 수정 지점]`
- 예시는 사용자가 말한 실제 자료명과 산출물명을 넣어 다시 작성합니다.

다음 단계로 뺄 것:

- 여러 파일 일괄 처리
- 실제 발송 자동화
- 디자인 완성도 높은 PPT 전체 제작

## 2. 데이터 정리/분석/대시보드

대표 상황:

- 이 프로젝트의 표, CSV, 시트, 목록형 데이터를 반복 정리합니다.
- 이 프로젝트의 계산, 분류, 필터 기준을 매번 사람이 다시 적용합니다.

반드시 묻는 질문:

- "오늘 넣어볼 원본 표 1개는 무엇인가요?"
- "사람이 지금 직접 계산하거나 분류하는 기준 1개는 무엇인가요?"
- "발표장에서 보여줄 핵심 지표나 표 1개는 무엇인가요?"

프로젝트 기반 MVP 예시 구성:

- `[이 프로젝트의 원본 표/CSV] -> [현장에서 바로 볼 지표나 분류표 1개]`
- `[이 프로젝트의 반복 계산 기준] -> [사람이 검토할 요약표]`
- 예시는 사용자가 말한 실제 컬럼, 계산 기준, 확인할 지표로 다시 작성합니다.

다음 단계로 뺄 것:

- 실시간 데이터베이스 연동
- 전체 BI 시스템
- 권한별 대시보드

## 3. 문의응대/매뉴얼/상담 보조

대표 상황:

- 이 프로젝트의 반복 질문, 기록, 기준 자료를 바탕으로 답변이나 안내를 만듭니다.
- 담당자가 이 프로젝트의 자료를 매번 찾아 문구를 다시 씁니다.

반드시 묻는 질문:

- "오늘 테스트할 실제 질문 예시 3개는 무엇인가요?"
- "답변이 참고해야 할 근거 자료는 무엇인가요?"
- "AI가 틀리면 위험한 답변 유형은 무엇이고, 사람이 어디서 검토해야 하나요?"

프로젝트 기반 MVP 예시 구성:

- `[이 프로젝트의 실제 질문/기록 묶음] -> [근거가 붙은 답변/안내 초안]`
- `[이 프로젝트의 기준 자료] -> [사람이 최종 확인할 절차나 문구]`
- 예시는 사용자가 말한 실제 질문 유형, 근거 자료, 위험한 답변 유형으로 다시 작성합니다.

다음 단계로 뺄 것:

- 공개 챗봇 배포
- 개인별 자동 상담
- 민감 사안 최종 판단 자동화

## 4. 분류/추천/매칭/리서치

대표 상황:

- 이 프로젝트의 여러 항목을 기준에 따라 분류, 추천, 매칭, 리서치합니다.
- 사람이 이 프로젝트에서 좋은 결과와 나쁜 결과를 구분하는 기준을 갖고 있습니다.

반드시 묻는 질문:

- "좋은 결과 예시 1개와 나쁜 결과 예시 1개는 무엇인가요?"
- "AI가 따라야 하는 분류, 추천, 매칭 기준은 무엇인가요?"
- "기준에 맞지 않거나 애매한 경우에는 어떻게 처리해야 하나요?"

프로젝트 기반 MVP 예시 구성:

- `[이 프로젝트의 분류/추천 대상 3-5건] -> [기준이 보이는 분류/추천표]`
- `[이 프로젝트의 애매한 사례] -> [보류 사유와 사람 검토 항목]`
- 예시는 사용자가 말한 실제 대상, 좋은 결과 기준, 나쁜 결과 기준으로 다시 작성합니다.

다음 단계로 뺄 것:

- 전체 웹 크롤링
- 외부 서비스 자동 등록
- 최종 선정 자동화

## 5. 웹앱/시스템 구축 욕구

대표 상황:

- 이 프로젝트의 업무를 화면이나 시스템 형태로 보여주고 싶어 합니다.
- 요구가 크지만 3시간 안에 전체 구현은 어렵습니다.

반드시 묻는 질문:

- "오늘 만들 화면 1개만 고르면 어떤 화면인가요?"
- "그 화면에서 사용자가 입력하거나 선택하는 값 1개는 무엇인가요?"
- "오늘은 수동으로 남길 부분과 다음 단계로 넘길 부분은 무엇인가요?"

프로젝트 기반 MVP 예시 구성:

- `[이 프로젝트의 사용자 입력 1개] -> [결과 확인 화면]`
- `[이 프로젝트의 담당자 행동 1개] -> [검토/수정 가능한 화면]`
- 예시는 사용자가 말한 실제 화면, 입력값, 사람이 확인할 결과로 다시 작성합니다.

다음 단계로 뺄 것:

- 로그인/권한
- 실제 DB 저장
- 결제/발송/외부 API 자동 실행

## 6. MVP 미정형

대표 상황:

- "아직 잘 모르겠다", "현실성 있는 것을 먼저", "조언을 받고 싶다", "플랫폼 구축"처럼 기능이 정해지지 않았습니다.
- 문제는 있지만 반복 업무나 산출물이 흐릿합니다.

반드시 묻는 질문:

- "가장 자주 반복되는 업무 1개는 무엇인가요?"
- "그 업무에서 오늘 사람이 덜 해도 되는 행동 1개는 무엇인가요?"
- "그 행동을 줄였다는 것을 보여줄 샘플 입력과 출력은 무엇인가요?"

프로젝트 기반 MVP 예시 구성:

- `[가장 자주 반복되는 이 프로젝트 업무] -> [오늘 볼 수 있는 결과물 1개]`
- `[사람이 덜 해도 되는 행동 1개] -> [입력과 출력이 보이는 작은 데모]`
- 예시는 사용자가 말한 실제 반복 업무와 줄이고 싶은 행동으로 다시 작성합니다.

다음 단계로 뺄 것:

- 조직 전체 업무 자동화
- 문제 탐색 컨설팅 전체
- 장기 제품 로드맵
