# 객관식 프롬프팅과 객관식 플래닝

객관식은 참가자의 답을 제한하기 위한 장치가 아니라, 빠르게 비교하고 합의하기 위한 수렴 장치입니다. 사용자가 이미 명확히 답한 내용은 객관식으로 다시 묻지 않습니다. 답이 추상적이거나 여러 방향이 섞였을 때만 사용합니다.

나쁜 객관식은 `추천안 1개 + 불필요한 들러리 2개`입니다. 좋은 객관식은 모두 선택해도 말이 되는 2-3개의 실행 가능한 대안입니다. 각 선택지는 같은 문제를 다른 관점으로 해결해야 하며, 사용자의 우선순위에 따라 실제로 선택이 갈릴 수 있어야 합니다.

선택지를 만들기 전에 사용자의 핵심 자동화 의도를 먼저 판별합니다. 사용자가 "AI가 산출물을 만들어주길" 기대한다면 모든 선택지는 자동 생성형이어야 합니다. 검수표, 체크리스트, 입력 보조는 독립 선택지가 아니라 자동 생성 결과를 안전하게 쓰기 위한 보강 요소로 둡니다.

## 두 가지 사용법

### 객관식 프롬프팅

하나의 빈칸을 빠르게 채울 때 씁니다.

- 문제 유형 선택
- 데모 산출물 형식 선택
- 사람 검토 방식 선택
- 제외 범위 선택
- 3시간 밖으로 보일 때 줄일 축 선택

형식:

```text
제가 보기엔 `[추천 선택지]`가 가장 맞아 보입니다.

A. [선택지 이름]: [한 줄 설명]
B. [선택지 이름]: [한 줄 설명]
C. [선택지 이름]: [한 줄 설명]

추천: [A/B/C]
이유: [현장 문제, 샘플 데이터, 3시간 제약을 근거로 한 한 문장]

A/B/C 중 고르거나, 섞고 싶은 부분을 말해주세요.
```

### 객관식 플래닝

MVP 방향 자체를 2-3개 완성안으로 비교할 때 씁니다.

- 여러 MVP 후보가 나왔을 때
- 사회혁신가가 "너무 단순하다"고 느낄 수 있을 때
- 시스템 구축 욕구를 3시간 데모로 줄여야 할 때
- 3시간 안/밖 판단이 `축소 필요`일 때

형식:

```text
3시간 MVP 후보를 세 가지로 자르면 이렇게 됩니다.

A. [안 이름]
- 입력: [샘플 입력 업무 묶음]
- 흐름: [2-4개 업무 단계]
- 산출물: [검토 가능한 최종 산출물]
- 장점: [현장 효용]
- 위험: [시간/데이터/검수 위험]
- 빠지는 것: [오늘 하지 않을 것]

B. [안 이름]
- 입력: [샘플 입력 업무 묶음]
- 흐름: [2-4개 업무 단계]
- 산출물: [검토 가능한 최종 산출물]
- 장점: [현장 효용]
- 위험: [시간/데이터/검수 위험]
- 빠지는 것: [오늘 하지 않을 것]

C. [안 이름]
- 입력: [샘플 입력 업무 묶음]
- 흐름: [2-4개 업무 단계]
- 산출물: [검토 가능한 최종 산출물]
- 장점: [현장 효용]
- 위험: [시간/데이터/검수 위험]
- 빠지는 것: [오늘 하지 않을 것]

추천: [A/B/C]
이유: [왜 이 안이 3시간 안에 가장 완결적인지]

어느 안으로 갈까요? A/B/C로 답해도 되고, 섞어서 조정해도 됩니다.
```

## 운영 규칙

- 선택지는 2-3개를 기본으로 하고, 4개를 넘기지 않습니다.
- 항상 추천안을 하나 제시합니다.
- 추천 이유는 한 문장으로 씁니다.
- 각 선택지는 서로 겹치지 않게 만듭니다.
- 명백히 약하거나 불필요한 선택지를 넣지 않습니다.
- 이미 제외하기로 한 기능, 3시간 밖의 시스템 구축, 단순 축소판은 선택지로 넣지 않습니다.
- 모든 선택지는 사용자의 핵심 자동화 의도를 같은 수준으로 충족해야 합니다.
- 사용자가 반복 제작물을 줄이고 싶어 하면 모든 선택지는 제작물 자동 생성형이어야 합니다.
- 사용자가 추천이나 매칭을 원하면 모든 선택지는 추천/매칭 결과를 포함해야 합니다.
- 사용자가 검수나 입력 오류 자체를 핵심 문제로 말한 경우에만 검수/입력 보조를 독립 선택지로 둡니다.
- 각 선택지는 서로 다른 성공 기준을 가져야 합니다. 예: 정확도, 현장 전달력, 운영 기록, 검수 안정성, 발표 설득력.
- 선택지마다 `장점`, `위험`, `빠지는 것`을 드러냅니다.
- 번호만 답해도 다음 단계로 진행합니다.
- 사용자가 선택지를 섞으면 새 합의안을 1-2문장으로 요약하고 진행합니다.
- 객관식이 길어지면 먼저 추천안만 제시하고, 사용자가 원할 때 다른 안을 펼칩니다.

## 선택지 품질 점검

객관식을 제시하기 전에 아래를 확인합니다.

- 사용자가 당연히 고를 선택지가 하나뿐이면 다시 만듭니다.
- 세 선택지가 모두 같은 문제를 해결하는가?
- 세 선택지가 모두 사용자의 핵심 자동화 기대를 충족하는가?
- 세 선택지가 모두 3시간 안에 데모 가능한가?
- 각 선택지가 서로 다른 장점과 위험을 갖는가?
- 선택하지 않은 안도 후속 후보나 다음 단계로 남길 만큼 의미가 있는가?
- 선택지가 기능 목록이 아니라 완성된 업무 흐름으로 표현됐는가?
- 검수/체크리스트/입력 보조가 자동 생성형 선택지를 밀어내고 있지 않은가?

## 단계별 선택지 예시

### 문제 유형

```text
제가 보기엔 A가 가장 맞아 보입니다.

A. 검토 보조형: 입력 묶음 -> 요약/분류 -> 근거 -> 담당자 검토표
B. 전달 산출물형: 입력 묶음 -> 초안 -> 공유 메시지/메일/안내문
C. 운영 정리형: 여러 건 입력 -> 우선순위표 -> 다음 행동 목록

추천: A
이유: 최종 판단을 자동화하지 않고도 담당자가 바로 검토할 근거를 만들 수 있기 때문입니다.

A/B/C 중 고르거나, 섞고 싶은 부분을 말해주세요.
```

### MVP 방향

```text
사용자가 반복 제작물 자동화를 원한다면 3시간 MVP 방향은 세 가지로 볼 수 있습니다.

A. 단일 산출물 자동 제작형
- 입력: 샘플 1-3건
- 흐름: 정보 추출 -> 핵심 정보 반영 -> 산출물 문구 생성 -> 제작 전 확인
- 산출물: 가장 중요한 문서나 메시지 1종의 초안
- 장점: 3시간 안에 완성도가 높고 자동화 체감이 분명합니다.
- 위험: 전체 업무 흐름 중 일부 산출물만 자동화합니다.
- 빠지는 것: 다른 제작물, 외부 발송, DB 저장

B. 복수 산출물 자동 제작형
- 입력: 샘플 1-3건
- 흐름: 정보 추출 -> 산출물별 문구 생성 -> 핵심 정보 일관성 확인 -> 제작용 초안 출력
- 산출물: 서로 연결된 문서나 제작물 2종의 초안
- 장점: 실제 반복 제작 업무를 더 크게 줄입니다.
- 위험: 산출물별 형식과 톤을 단순하게 유지해야 합니다.
- 빠지는 것: 실제 디자인 완성, 외부 발송, DB 저장

C. 제작 + 커뮤니케이션 자동 생성형
- 입력: 샘플 1-3건
- 흐름: 정보 추출 -> 제작물 초안 생성 -> 안내/공유 문구 생성 -> 제작 전 확인
- 산출물: 제작물 초안, 후속 안내 메시지, 내부 공유 문구
- 장점: 제작 이후 커뮤니케이션까지 한 번에 이어집니다.
- 위험: 핵심 제작물의 완성도가 낮아지지 않도록 범위를 조절해야 합니다.
- 빠지는 것: 실제 발송, 입금 처리, 외부 시스템 자동 입력

추천: B
이유: 자동화 체감이 크면서도 3시간 안에 서로 연결된 제작물 2종을 안정적으로 보여줄 수 있기 때문입니다.

어느 안으로 갈까요?
```

### 3시간 밖일 때 축소 선택

```text
지금 범위는 3시간을 넘을 가능성이 큽니다. 줄이는 방법은 세 가지입니다.

A. 입력 줄이기: 전체 데이터를 샘플 3-5건으로 제한
B. 단계 줄이기: 자동 실행 대신 검토 가능한 표와 메시지만 생성
C. 연동 빼기: API/DB/발송 없이 수동 업로드와 복사용 결과로 시연

추천: C
이유: 연동을 빼도 현장 업무의 핵심 판단과 산출물은 보여줄 수 있기 때문입니다.

A/B/C 중 무엇을 줄일까요?
```
