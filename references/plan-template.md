# PLAN.md 템플릿

`PLAN.md`는 아래 구조를 정확히 사용합니다.

```markdown
---
team_name: "[팀 이름]"
project_name: "[프로젝트 이름]"
created_date: "[YYYY-MM-DD]"
event: "Codex Impact Workshop"
version: "1"
---

# [프로젝트 이름] 실행 계획

## 팀
- 사회혁신가: [이름]
- 개발자: [이름]

## 문제
- 실제 업무 문제: [내용]
- 가장 어려움을 겪는 사람: [내용]
- 문제가 발생하는 순간: [내용]

## 현재 업무 흐름
- 입력 자료: [내용]
- 처리 순서: [내용]
- 판단 기준: [내용]
- 현재 산출물: [내용]
- 현재 사용 도구: [내용]

## 기대 변화
- 기대 변화: [내용]
- 관찰 가능한 개선: [내용]

## 3시간 MVP
- 해결할 좁은 문제: [3시간 안에 끝까지 해결할 실제 업무 문제]
- 핵심 기능 1개: [내용]
- 포함할 업무 단계: [예: 추출 -> 분류 -> 근거 표시 -> 검토 체크리스트]
- 데모 입력: [내용]
- 데모 산출물: [내용]
- 성공 기준: [내용]
- 3시간 안/밖 판단: [안/축소 필요/밖 - 근거]

## 제외 범위
- 오늘 하지 않을 것: [내용]
- 다음 단계 아이디어: [내용]

## 3시간 구현 계획
- 0-30분: [내용]
- 30-90분: [내용]
- 90-150분: [내용]
- 150-180분: [내용]

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: [핵심 기능이 끝난 뒤 바로 이어서 할 작업]
- 후보 1 시작 조건: [언제 이 후보로 넘어갈지]
- 후보 1 구현 내용: [구체적인 구현 단위]
- 후보 1 데모 산출물: [발표장에서 추가로 보여줄 결과]
- 후보 1 예상 추가 시간: [예: 20분/40분/60분]
- 후보 1 검수 기준: [성공 여부를 확인하는 기준]
- 후보 2 작업: [핵심 기능이 끝난 뒤 바로 이어서 할 작업]
- 후보 2 시작 조건: [언제 이 후보로 넘어갈지]
- 후보 2 구현 내용: [구체적인 구현 단위]
- 후보 2 데모 산출물: [발표장에서 추가로 보여줄 결과]
- 후보 2 예상 추가 시간: [예: 20분/40분/60분]
- 후보 2 검수 기준: [성공 여부를 확인하는 기준]
```

정보가 없으면 `미정`, 해당하지 않으면 `해당 없음`이라고 씁니다.
