# PLAN.md 템플릿

`PLAN.md`는 아래 구조를 정확히 사용합니다.

```markdown
---
team_name: "[팀 이름]"
project_name: "[프로젝트 이름]"
created_date: "[YYYY-MM-DD]"
event: "Codex Impact Workshop"
version: "1"
---

# [프로젝트 이름] 실행 계획

## 팀
- 사회혁신가: [이름]
- 개발자: [이름]

## 문제
- 실제 업무 문제: [내용]
- 가장 어려움을 겪는 사람: [내용]
- 문제가 발생하는 순간: [내용]

## 현재 업무 흐름
- 입력 자료: [내용]
- 처리 순서: [내용]
- 판단 기준: [내용]
- 현재 산출물: [내용]
- 현재 사용 도구: [내용]

## 기대 변화
- 기대 변화: [내용]
- 관찰 가능한 개선: [내용]

## 3시간 MVP
- 핵심 기능 1개: [내용]
- 데모 입력: [내용]
- 데모 산출물: [내용]
- 성공 기준: [내용]

## 제외 범위
- 오늘 하지 않을 것: [내용]
- 다음 단계 아이디어: [내용]

## 3시간 구현 계획
- 0-30분: [내용]
- 30-90분: [내용]
- 90-150분: [내용]
- 150-180분: [내용]
```

정보가 없으면 `미정`, 해당하지 않으면 `해당 없음`이라고 씁니다.
