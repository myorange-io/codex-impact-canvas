# 후원 미납 위험 목록 생성 실행 계획

## 팀
- 사회혁신가: 이명희
- 개발자: 조성도

## 문제
- 실제 업무 문제: 후원자 관리 과정에서 후원 내역 CSV를 사람이 확인하며 미납 또는 중단 위험 대상을 찾기 어렵다.
- 가장 어려움을 겪는 사람: 후원자 관리를 담당하는 실무자와 지속적인 관계 관리가 필요한 후원자
- 문제가 발생하는 순간: 월별 후원 내역을 확인하고 미납 대상자에게 연락해야 할 때

## 현재 업무 흐름
- 입력 자료: 후원 내역 CSV
- 처리 순서: 후원 내역 다운로드 -> 후원자별 최근 납부월 확인 -> 최근 2개월 미납 여부 판단 -> 연락 대상 목록 작성
- 판단 기준: 최근 2개월 동안 정상 납부 기록이 없는지 여부
- 현재 산출물: 미납/중단 위험 대상자 목록
- 현재 사용 도구: CSV 파일, 스프레드시트

## 기대 변화
- 기대 변화: 담당자가 CSV를 반복 확인하며 최근 2개월 미납자를 찾는 시간을 줄인다.
- 관찰 가능한 개선: 후원 내역 CSV 1개를 넣으면 최근 2개월 미납자가 빠짐없이 표시된 목록이 나온다.

## 3시간 MVP
- 해결할 좁은 문제: 후원 내역 CSV에서 최근 2개월 미납자를 담당자가 검토 가능한 목록으로 만든다.
- 핵심 기능 1개: 후원 내역 CSV 업로드 -> 최근 2개월 미납/중단 위험 목록 자동 생성
- 포함할 업무 단계: CSV 업로드 -> 납부월 확인 -> 최근 2개월 미납자 판별 -> 위험 목록 생성 -> 담당자 최종 검토
- 데모 입력: 익명화한 후원 내역 CSV 1개
- 데모 산출물: 최근 2개월 미납/중단 위험 목록
- 성공 기준: 최근 2개월 미납자가 빠짐없이 표시되면 성공이고, 정상 납부자가 미납자로 표시되면 실패다.
- 3시간 안/밖 판단: 안 - CSV 수동 업로드와 목록 생성만 포함하고 실제 연락/발송 자동화는 제외한다.

## 제외 범위
- 오늘 하지 않을 것: 후원자 전체 관리 시스템, 외부 CRM/결제 시스템 연동, 실제 문자/메일 발송 자동화, 최종 연락 대상 자동 확정
- 다음 단계 아이디어: 연락 우선순위 표시, 안내 문자 초안 생성, 월별 후원 추이 대시보드

## 3시간 구현 계획
- 0-30분: 익명화한 후원 내역 CSV 샘플과 최근 2개월 미납 판정 기준을 확인한다.
- 30-90분: CSV를 읽어 후원자별 최근 납부월을 계산하고 최근 2개월 미납 여부를 판별하는 처리 흐름을 만든다.
- 90-150분: 미납/중단 위험 목록을 표 형식으로 출력하고 담당자 검토 칸을 붙인다.
- 150-180분: 샘플 CSV로 정상 납부자 오탐과 미납자 누락 여부를 검수하고 발표 흐름을 정리한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 연락 우선순위 표시
- 후보 1 시작 조건: 최근 2개월 미납자 목록 생성이 정상 동작할 때
- 후보 1 구현 내용: 미납 개월 수, 누적 후원액, 장기 후원 여부를 기준으로 높음/보통/낮음 우선순위를 표시한다.
- 후보 1 데모 산출물: 우선순위가 붙은 미납/중단 위험 목록
- 후보 1 예상 추가 시간: 30-40분
- 후보 1 검수 기준: 담당자가 누구부터 확인할지 바로 판단할 수 있다.
- 후보 2 작업: 안내 문자 초안 생성
- 후보 2 시작 조건: 미납 대상자의 이름, 최근 후원월, 미납 개월 수가 목록에 포함될 때
- 후보 2 구현 내용: 대상자별로 사람이 검토할 수 있는 정중한 안내 문자 초안을 생성한다.
- 후보 2 데모 산출물: 미납자 3명에 대한 문자 초안
- 후보 2 예상 추가 시간: 30-40분
- 후보 2 검수 기준: 바로 발송하지 않고 담당자가 수정/승인할 수 있는 수준의 문구가 나온다.
