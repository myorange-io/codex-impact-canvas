# 후원 미납 위험 목록 생성 워크플로 분석

## 라이브러리 메타데이터
- 사례 ID: impact-agent-20260610-donor-payment-risk-list
- 분야: 비영리/후원관리
- 업무 유형: 데이터정리/위험목록생성
- 에이전트 역할: 분류/검토보조
- 데이터 특성: 후원자별 납부일, 납부월, 금액, 상태가 포함될 수 있는 후원 내역 표
- 정형/비정형 여부: 정형
- 개인정보 포함 여부: 개인정보 포함. 후원자 이름, 연락처, 결제/후원 이력은 익명화 필요
- 연결 수준: 수동 업로드
- MCP/CLI 연결 여부: 사용 안 함
- 사람 검토: 최종 검토
- 재사용 가능성: 템플릿 재사용 가능
- 산출물 형식: md

## 기존 업무 흐름
- 업무 시작 조건: 월별 후원 내역 확인 또는 미납자 연락 준비
- 데이터 수: 데모 CSV 1개, 실제 데이터 수는 미정
- 반복 빈도: 매월 또는 후원 내역 정리 시 반복
- 기존 소요 시간: 미정
- 입력 데이터 출처: 후원 관리 시스템 또는 결제 내역에서 내려받은 CSV
- 기존 단계: 담당자가 CSV를 열고 후원자별 최근 납부 기록을 확인한 뒤 미납 대상자를 별도 목록으로 옮긴다.
- 기존 산출물: 수동으로 정리한 미납/중단 위험 대상자 목록
- 불편한 지점: 반복 확인 시간이 길고, 미납자 누락이나 정상 납부자 오탐 가능성이 있다.

## 자동화 대상
- 선정한 업무: 후원 내역 CSV에서 최근 2개월 미납자를 찾아 위험 목록을 생성하는 일
- 선정 이유: 입력과 출력이 명확하고 발표장에서 샘플 CSV 1개로 성공/실패를 바로 검수할 수 있다.
- 제외한 업무: 후원자 전체 관리 자동화, 연락 자동 발송, 외부 시스템 실시간 연동

## AI 에이전트 설계
- 에이전트 입력 조건: 후원자 식별값과 월별 납부 기록이 들어 있는 익명화 CSV 1개
- 에이전트 산출물 조건: 최근 2개월 미납자로 판정된 후원자 목록과 담당자 검토용 근거
- 판단 경계: AI는 최근 2개월 미납 후보를 산출하고, 담당자가 실제 미납 여부와 예외 사유를 최종 확인한다.
- 사용 도구: Codex, CSV 처리 스크립트 또는 스프레드시트
- 바뀐 업무 흐름: 익명화 CSV 업로드 -> AI/스크립트 미납 판별 -> 위험 목록 생성 -> 담당자 최종 검토 -> 연락 여부 결정

## 사람 검토와 개입
- 주요 검수 포인트: 이름/식별값, 연락처, 실제 미납 여부, 예외 사유, 정상 납부자 오탐 여부를 확인한다.
- 실패 시 사람이 이어받는 방식: 결제월이 불명확하거나 CSV 형식이 맞지 않으면 담당자가 원본 CSV를 열어 기존 방식으로 확인한다.
- 예상 오류 유형: 정상 납부자 오탐, 최근 납부월 계산 오류, CSV 컬럼 해석 오류, 예외 후원자 누락
- 평가 샘플: 익명화한 후원 내역 CSV 1개와 담당자가 알고 있는 최근 2개월 미납자 기대 목록

## 데이터와 접근
- 개인정보 처리: 발표와 사례 공개에는 이름, 연락처, 계좌/결제 식별자, 실제 후원 금액을 제거하거나 더미값으로 바꾼다.
- 권한/접근 이슈: 원본 후원 내역 CSV는 담당자만 접근 가능하다고 가정한다.
- 결과물의 공유 가능 범위: 익명화 후 공개 가능

## 영향과 재사용
- 기존 대비 소요 시간 변화: 미정. 샘플 CSV 처리 후 수동 확인 시간과 자동 목록 생성 시간을 비교한다.
- 품질 변화: 최근 2개월 미납자 누락 방지와 정상 납부자 오탐 감소를 기대한다.
- 재사용 방법: 미납 판정 기준과 CSV 컬럼명을 기관별 후원 데이터 구조에 맞게 바꾸면 다른 후원관리 업무에도 재사용할 수 있다.
- 유지관리 주체: 후원자 관리 담당자
- 다음 개선: 연락 우선순위 표시와 안내 문자 초안 생성
