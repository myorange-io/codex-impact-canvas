- 정형/비정형 여부: 비정형

## 기존 업무 흐름
- 업무 시작 조건: 신청서 접수
- 데이터 수: 데모 1건, 실제 업무 주 20-40건
- 반복 빈도: 매주 신청 접수 후 반복
- 기존 소요 시간: 건당 8-12분
- 입력 데이터 출처: 구글폼과 상담 메모
- 기존 단계: 담당자가 신청서와 상담 메모를 읽고 스프레드시트에 긴급도를 적는다.
- 기존 산출물: 회의용 우선순위 목록
- 불편한 지점: 반복 읽기와 근거 누락

## 자동화 대상
- 선정한 업무: 신청서/상담 메모 요약과 긴급도 초안 작성
- 선정 이유: 샘플 1건으로 입력과 산출물을 명확히 보여줄 수 있다.
- 제외한 업무: 지원 대상 자동 결정과 발송

## AI 에이전트 설계
- 에이전트 입력 조건: 개인정보를 제거한 신청서와 상담 메모 텍스트
- 에이전트 산출물 조건: 긴급도 초안, 근거 요약, 확인 질문
- 판단 경계: AI는 초안을 만들고 담당자가 최종 지원 여부를 결정한다.
- 사용 도구: Codex, ChatGPT
- 바뀐 업무 흐름: 익명화 텍스트 입력 -> AI 초안 생성 -> 담당자 검토 -> 회의 자료 반영

## 사람 검토와 개입
- 주요 검수 포인트: 긴급도 근거가 충분한지, 개인정보가 남아 있지 않은지, 최종 지원 판단이 사람에게 남아 있는지 확인
- 실패 시 사람이 이어받는 방식: 근거가 부족하면 기존 방식으로 상담 메모를 다시 읽는다.
- 예상 오류 유형: 긴급도 과대평가, 맥락 누락, 민감정보 포함 위험
- 평가 샘플: 익명화 신청서 3건과 담당자가 정한 기대 긴급도

## 데이터와 접근
- 개인정보 처리: 이름, 연락처, 주소를 제거하고 가구 특성만 남긴다.
- 권한/접근 이슈: 원본 상담 메모 접근은 담당자만 가능
- 결과물의 공유 가능 범위: 익명화 후 공개 가능

## 영향과 재사용
- 기존 대비 소요 시간 변화: 건당 8-12분에서 초안 확인 3분 내외
- 품질 변화: 회의 전 근거 요약의 일관성 향상
- 재사용 방법: 긴급도 기준과 확인 질문을 기관별로 바꾼다.
- 유지관리 주체: 복지기관 사업 담당자
- 다음 개선: 일괄 처리와 검토 화면
