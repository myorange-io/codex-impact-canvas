# 긴급식품지원 신청 검토 보조 실행 계획

## 팀
- 사회혁신가: 김사회
- 개발자: 박개발

## 문제
- 실제 업무 문제: 긴급식품지원 신청서와 상담 메모를 사람이 반복해서 읽고 지원 우선순위를 정리한다.
- 가장 어려움을 겪는 사람: 복지기관 신청 접수 담당자와 식품 지원이 필요한 가구
- 문제가 발생하는 순간: 월요일 오전처럼 주말 동안 접수된 신청이 한꺼번에 몰릴 때

## 현재 업무 흐름
- 입력 자료: 익명화한 신청서 텍스트와 상담 메모
- 처리 순서: 신청 접수 -> 상담 메모 확인 -> 긴급도 표시 -> 담당자 회의용 요약 작성
- 판단 기준: 가구 상황, 식사 공백 기간, 아동/노인 동거 여부, 기존 지원 여부
- 현재 산출물: 긴급도별 신청자 목록과 회의용 요약
- 현재 사용 도구: 구글폼, 스프레드시트, 상담 메모 문서

## 기대 변화
- 기대 변화: 담당자가 반복해서 읽는 시간을 줄이고 회의 전에 우선순위 초안을 빠르게 만든다.
- 관찰 가능한 개선: 샘플 신청 3건에서 긴급도, 근거 문장, 담당자 확인 필요 항목이 회의용 표로 나온다.

## 3시간 MVP
- 해결할 좁은 문제: 월요일 오전 회의 전에 신청 3건을 담당자가 검토 가능한 우선순위 표로 만든다.
- 핵심 기능 1개: 익명화한 신청서와 상담 메모 3건을 넣으면 긴급도 우선순위표, 근거 요약, 확인 질문, 검토 체크리스트를 생성한다.
- 포함할 업무 단계: 신청 내용 요약 -> 긴급도 분류 -> 근거 표시 -> 확인 질문 생성 -> 담당자 검토 체크리스트
- 데모 입력: 개인정보를 제거한 신청서/상담 메모 3건 묶음
- 데모 산출물: 긴급도 우선순위표, 건별 근거 요약, 담당자 확인 질문, 검토 체크리스트
- 성공 기준: 샘플 3건 모두 긴급도와 근거가 표시되고 담당자가 회의에서 바로 확인할 질문을 볼 수 있다.
- 3시간 안/밖 판단: 안 - 익명화 텍스트와 수동 업로드만 필요하고 최종 지원 판단은 담당자가 한다.

## 제외 범위
- 오늘 하지 않을 것: 지원 대상 자동 확정, 문자 발송
- 다음 단계 아이디어: 여러 신청서 일괄 처리와 담당자 배정

## 3시간 구현 계획
- 0-30분: 샘플 입력과 긴급도 기준을 정한다.
- 30-90분: 요약/분류 프롬프트와 출력 형식을 만든다.
- 90-150분: 간단한 CLI 또는 폼 기반 데모를 만든다.
- 150-180분: 샘플 3건으로 결과를 검토하고 발표 자료를 정리한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 회의 진행용 브리핑 문단 추가
- 후보 1 시작 조건: 우선순위표와 검토 체크리스트가 통과하고 35분 이상 남았을 때
- 후보 1 구현 내용: 표 결과를 바탕으로 담당자가 회의에서 읽을 5문장 브리핑과 결정 보류 사유를 생성한다.
- 후보 1 데모 산출물: 회의 시작용 브리핑 문단과 보류 사유 목록
- 후보 1 예상 추가 시간: 45분
- 후보 1 검수 기준: 담당자가 표를 다시 읽지 않아도 먼저 논의할 신청과 보류 사유를 이해할 수 있다.
- 후보 2 작업: 담당자 배정 초안 추가
- 후보 2 시작 조건: 긴급도 분류가 안정되고 30분 이상 남았을 때
- 후보 2 구현 내용: 긴급도와 확인 질문을 기준으로 담당자 배정 후보와 배정 근거를 붙인다.
- 후보 2 데모 산출물: 건별 담당자 배정 후보가 포함된 회의용 표
- 후보 2 예상 추가 시간: 30분
- 후보 2 검수 기준: 담당자가 최종 배정 전 확인할 후보와 근거를 바로 볼 수 있다.
