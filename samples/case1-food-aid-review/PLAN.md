# 긴급식품지원 신청 검토 보조 실행 계획

## 팀
- 사회혁신가: 김사회
- 개발자: 박개발

## 문제
- 실제 업무 문제: 긴급식품지원 신청서와 상담 메모를 사람이 반복해서 읽고 지원 우선순위를 정리한다.
- 가장 어려움을 겪는 사람: 복지기관 신청 접수 담당자와 식품 지원이 필요한 가구
- 문제가 발생하는 순간: 월요일 오전처럼 주말 동안 접수된 신청이 한꺼번에 몰릴 때

## 자료 확인
- 자료 상태: 익명화 샘플
- 샘플 입력: 이름, 연락처, 주소를 제거한 신청서와 상담 메모 1건
- 개인정보 처리: 이름, 연락처, 주소를 제거하고 가구 특성만 남긴다.
- 자료 유효성 확인: 긴급도 판단 기준과 확인 질문을 만들 수 있는 최소 상담 맥락이 남아 있다.

## 현재 업무 흐름
- 입력 자료: 익명화한 신청서 텍스트와 상담 메모
- 처리 순서: 신청 접수 -> 상담 메모 확인 -> 긴급도 표시 -> 담당자 회의용 요약 작성
- 판단 기준: 가구 상황, 식사 공백 기간, 아동/노인 동거 여부, 기존 지원 여부
- 현재 산출물: 긴급도별 신청자 목록과 회의용 요약
- 현재 사용 도구: 구글폼, 스프레드시트, 상담 메모 문서

## 결과물 형태
- 문제 유형: 분류/추천/매칭/리서치
- 선택한 결과물 형태: 자동화 도구
- 선택 이유: 익명화 텍스트를 넣으면 담당자가 검토할 긴급도와 근거 요약을 바로 만들 수 있다.

## 기대 변화
- 기대 변화: 담당자가 반복해서 읽는 시간을 줄이고 회의 전에 우선순위 초안을 빠르게 만든다.
- 관찰 가능한 개선: 샘플 신청 1건에서 긴급도, 근거 문장, 담당자 확인 필요 항목이 나온다.

## 3시간 MVP
- 핵심 기능 1개: 익명화한 신청서와 상담 메모를 넣으면 긴급도 초안과 근거 요약을 생성한다.
- 데모 입력: 개인정보를 제거한 신청서/상담 메모 1건
- 데모 산출물: 긴급도, 근거 요약, 담당자 확인 질문 목록
- AI 역할: 검토보조
- 사람 검토 지점: 담당자가 긴급도 근거와 개인정보 제거 여부를 회의 전 최종 확인한다.
- 성공 기준: 샘플 1건에서 사람이 회의 전에 검토할 수 있는 긴급도 초안과 근거 2개 이상이 나온다.
- 실패 신호: 긴급도 근거가 부족하거나 개인정보가 결과에 남아 있으면 실패로 본다.

## 제외 범위
- 오늘 하지 않을 것: 지원 대상 자동 확정, 문자 발송
- 다음 단계 아이디어: 여러 신청서 일괄 처리와 담당자 배정

## 3시간 구현 계획
- 0-30분: 샘플 입력과 긴급도 기준을 정한다.
- 30-90분: 요약/분류 프롬프트와 출력 형식을 만든다.
- 90-150분: 간단한 CLI 또는 폼 기반 데모를 만든다.
- 150-180분: 샘플 3건으로 결과를 검토하고 발표 자료를 정리한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 신청서 3건 일괄 처리 결과표 추가
- 후보 1 시작 조건: 핵심 기능이 샘플 1건에서 통과하고 45분 이상 남았을 때
- 후보 1 구현 내용: 신청서/상담 메모 여러 건을 구분자로 나눠 같은 프롬프트를 반복 적용하고 결과를 표로 묶는다.
- 후보 1 데모 산출물: 신청 3건의 긴급도, 근거, 확인 질문 요약표
- 후보 1 예상 추가 시간: 45분
- 후보 1 검수 기준: 3건 모두에 긴급도와 근거가 빠짐없이 표시되고 담당자가 우선순위를 비교할 수 있다.
- 후보 2 작업: 담당자 검토 체크리스트 추가
- 후보 2 시작 조건: 핵심 기능 출력 형식이 안정되고 30분 이상 남았을 때
- 후보 2 구현 내용: 각 신청 결과 끝에 개인정보 제거, 근거 충분성, 최종 판단 보류 여부를 확인하는 체크리스트를 붙인다.
- 후보 2 데모 산출물: 검토 체크리스트가 포함된 회의용 요약
- 후보 2 예상 추가 시간: 30분
- 후보 2 검수 기준: 담당자가 지원 여부를 확정하기 전에 확인할 항목을 바로 볼 수 있다.
