# 멘토링 일지 위험 신호 요약 실행 계획

## 팀
- 사회혁신가: 이현장
- 개발자: 최개발

## 문제
- 실제 업무 문제: 청소년 멘토링 일지에서 긴급 확인이 필요한 신호를 사람이 늦게 발견한다.
- 가장 어려움을 겪는 사람: 멘토링 코디네이터와 참여 청소년
- 문제가 발생하는 순간: 주간 일지가 한꺼번에 제출되는 금요일 오후

## 현재 업무 흐름
- 입력 자료: 멘토가 작성한 주간 일지 텍스트
- 처리 순서: 일지 수집 -> 코디네이터 읽기 -> 위험 신호 표시 -> 담당자 공유
- 판단 기준: 결석, 정서 위기 표현, 가정 문제, 상담 필요 신호
- 현재 산출물: 확인 필요 일지 목록과 담당자 메모
- 현재 사용 도구: 구글문서, 스프레드시트

## 기대 변화
- 기대 변화: 확인 필요 일지를 먼저 볼 수 있어 대응 지연을 줄인다.
- 관찰 가능한 개선: 샘플 일지 1건에서 확인 필요 신호와 후속 질문 초안이 나온다.

## 3시간 MVP
- 핵심 기능 1개: 멘토링 일지 텍스트를 넣으면 확인 필요 신호와 후속 질문 초안을 만든다.
- 데모 입력: 익명화한 주간 일지 1건
- 데모 산출물: 위험 신호 요약, 근거 문장, 후속 확인 질문
- 성공 기준: 샘플 일지의 핵심 위험 신호를 놓치지 않고 사람이 확인할 질문 2개 이상을 제안한다.

## 제외 범위
- 오늘 하지 않을 것: 상담 판정, 보호자 연락 자동화
- 다음 단계 아이디어: 주간 일지 일괄 분석

## 3시간 구현 계획
- 0-30분: 위험 신호 기준과 샘플 일지를 정리한다.
- 30-90분: 분류/요약 프롬프트와 출력 형식을 만든다.
- 90-150분: 텍스트 입력 데모를 구현한다.
- 150-180분: 코디네이터 관점으로 결과를 검수한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 주간 일지 5건 일괄 요약 추가
- 후보 1 시작 조건: 샘플 1건에서 위험 신호와 후속 질문이 안정적으로 나오고 50분 이상 남았을 때
- 후보 1 구현 내용: 여러 일지를 번호로 구분해 입력받고 확인 필요 순서대로 요약표를 만든다.
- 후보 1 데모 산출물: 일지 5건의 위험 신호 수준, 근거 문장, 후속 질문 목록
- 후보 1 예상 추가 시간: 50분
- 후보 1 검수 기준: 확인 필요 일지가 표 상단에 오고 각 일지마다 근거 문장이 1개 이상 나온다.
- 후보 2 작업: 코디네이터 공유용 알림 문구 추가
- 후보 2 시작 조건: 핵심 요약 결과가 통과하고 25분 이상 남았을 때
- 후보 2 구현 내용: 위험 신호 요약을 바탕으로 내부 공유용 짧은 알림 문구와 확인 요청 문장을 생성한다.
- 후보 2 데모 산출물: 담당자에게 보낼 내부 공유 메시지 초안
- 후보 2 예상 추가 시간: 25분
- 후보 2 검수 기준: 민감한 원문 없이도 담당자가 무엇을 확인해야 하는지 이해할 수 있다.
