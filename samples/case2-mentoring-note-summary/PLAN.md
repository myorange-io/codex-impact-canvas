# 멘토링 일지 위험 신호 요약 실행 계획

## 팀
- 사회혁신가: 이현장
- 개발자: 최개발

## 문제
- 실제 업무 문제: 청소년 멘토링 일지에서 긴급 확인이 필요한 신호를 사람이 늦게 발견한다.
- 가장 어려움을 겪는 사람: 멘토링 코디네이터와 참여 청소년
- 문제가 발생하는 순간: 주간 일지가 한꺼번에 제출되는 금요일 오후

## 현재 업무 흐름
- 입력 자료: 멘토가 작성한 주간 일지 텍스트
- 처리 순서: 일지 수집 -> 코디네이터 읽기 -> 위험 신호 표시 -> 담당자 공유
- 판단 기준: 결석, 정서 위기 표현, 가정 문제, 상담 필요 신호
- 현재 산출물: 확인 필요 일지 목록과 담당자 메모
- 현재 사용 도구: 구글문서, 스프레드시트

## 기대 변화
- 기대 변화: 확인 필요 일지를 먼저 볼 수 있어 대응 지연을 줄인다.
- 관찰 가능한 개선: 샘플 일지 5건에서 확인 필요 순서, 근거, 후속 질문, 내부 공유 문구가 나온다.

## 3시간 MVP
- 해결할 좁은 문제: 금요일 오후 제출된 주간 일지 5건을 코디네이터가 먼저 확인할 순서로 정리한다.
- 핵심 기능 1개: 익명화한 멘토링 일지 5건을 넣으면 확인 필요 우선순위표, 근거 문장, 후속 질문, 내부 공유 문구를 생성한다.
- 포함할 업무 단계: 일지 요약 -> 위험 신호 표시 -> 확인 필요 순서 정렬 -> 후속 질문 생성 -> 내부 공유 문구 작성
- 데모 입력: 익명화한 주간 일지 5건 묶음
- 데모 산출물: 확인 필요 우선순위표, 위험 신호 근거, 후속 질문, 코디네이터 공유 메시지
- 성공 기준: 확인 필요 일지가 표 상단에 오고 각 일지마다 근거 문장과 사람이 확인할 질문이 1개 이상 나온다.
- 3시간 안/밖 판단: 안 - 익명화 텍스트 묶음으로 시연 가능하고 상담 판정은 코디네이터에게 남긴다.

## 제외 범위
- 오늘 하지 않을 것: 상담 판정, 보호자 연락 자동화
- 다음 단계 아이디어: 주간 일지 일괄 분석

## 3시간 구현 계획
- 0-30분: 위험 신호 기준과 샘플 일지를 정리한다.
- 30-90분: 분류/요약 프롬프트와 출력 형식을 만든다.
- 90-150분: 텍스트 입력 데모를 구현한다.
- 150-180분: 코디네이터 관점으로 결과를 검수한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 민감 표현 제거 점검 추가
- 후보 1 시작 조건: 우선순위표와 공유 문구가 통과하고 40분 이상 남았을 때
- 후보 1 구현 내용: 내부 공유 문구에서 청소년 식별 정보와 민감 원문을 제거했는지 점검하는 체크 항목을 붙인다.
- 후보 1 데모 산출물: 민감 표현 제거 체크가 포함된 내부 공유 메시지
- 후보 1 예상 추가 시간: 50분
- 후보 1 검수 기준: 담당자에게 공유해도 원문 민감정보가 노출되지 않는다.
- 후보 2 작업: 담당자별 확인 요청 분리
- 후보 2 시작 조건: 확인 필요 순서가 안정되고 30분 이상 남았을 때
- 후보 2 구현 내용: 코디네이터, 멘토, 상담 담당자가 각각 확인할 질문을 분리해 표시한다.
- 후보 2 데모 산출물: 역할별 확인 요청 목록
- 후보 2 예상 추가 시간: 25분
- 후보 2 검수 기준: 각 담당자가 자신이 확인할 항목만 바로 볼 수 있다.
