# 멘토링 일지 위험 신호 요약 워크플로 분석

## 라이브러리 메타데이터
- 사례 ID: impact-agent-20260605-mentoring-risk-summary
- 분야: 교육
- 업무 유형: 요약
- 에이전트 역할: 검토보조
- 데이터 특성: 멘토가 자유롭게 작성한 주간 일지 텍스트
- 정형/비정형 여부: 비정형
- 개인정보 포함 여부: 민감정보 포함. 청소년 식별 정보와 상담 내용 제거 필요
- 연결 수준: 수동 업로드
- MCP/CLI 연결 여부: MCP/CLI 연결 없음. 익명화 일지 텍스트 수동 업로드
- 사람 검토: 중간 검토
- 재사용 가능성: 분야 특화
- 산출물 형식: md

## 기존 업무 흐름
- 업무 시작 조건: 주간 일지 제출
- 데이터 수: 데모 5건, 실제 업무 주 30-60건
- 반복 빈도: 매주 금요일 일지 제출 후 반복
- 기존 소요 시간: 일지당 5분
- 입력 데이터 출처: 멘토 작성 구글문서
- 기존 단계: 코디네이터가 모든 일지를 읽고 확인 필요 항목을 표시한다.
- 기존 산출물: 확인 필요 일지 목록
- 불편한 지점: 많은 일지 속 긴급 신호 발견 지연

## 자동화 대상
- 선정한 업무: 일지 5건 확인 필요 우선순위표와 내부 공유 문구 작성
- 선정 이유: 결정은 사람이 하되 주간 일지 묶음에서 먼저 볼 대상을 끝까지 정리할 수 있다.
- 제외한 업무: 상담 필요 여부 최종 판단

## AI 에이전트 설계
- 에이전트 입력 조건: 이름을 제거한 일지 텍스트 5건
- 에이전트 산출물 조건: 확인 필요 우선순위표, 위험 신호 근거, 후속 질문, 내부 공유 문구
- 판단 경계: AI는 위험 가능성을 표시하고 코디네이터가 최종 판단한다.
- 사용 도구: Codex, ChatGPT
- 바뀐 업무 흐름: 익명화 일지 입력 -> AI 요약 -> 코디네이터 우선 검토 -> 담당자 공유

## 사람 검토와 개입
- 주요 검수 포인트: 위험 신호가 과장되지 않았는지, 원문 맥락이 누락되지 않았는지, 공유 전 민감 표현이 제거됐는지 확인
- 실패 시 사람이 이어받는 방식: AI가 불확실하다고 표시하면 원문 일지를 사람이 읽는다.
- 예상 오류 유형: 위험 신호 누락, 과잉 경고, 민감 표현 노출
- 평가 샘플: 익명화 일지 5건과 코디네이터 기대 표시

## 데이터와 접근
- 개인정보 처리: 이름, 학교, 연락처 등 식별 정보를 제거한다.
- 권한/접근 이슈: 민감정보라 원문 공유 불가
- 결과물의 공유 가능 범위: 내부 공유만

## 영향과 재사용
- 기존 대비 소요 시간 변화: 전체 일지 선독 시간을 줄이고 우선 검토 대상을 먼저 확인
- 품질 변화: 위험 신호 검토 기준의 일관성 향상
- 재사용 방법: 위험 신호 기준과 후속 질문 문구를 프로그램별로 바꾼다.
- 유지관리 주체: 멘토링 코디네이터
- 다음 개선: 주간 일지 일괄 업로드와 태그 통계
