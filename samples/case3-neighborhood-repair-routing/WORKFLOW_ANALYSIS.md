# 생활 불편 제보 분류와 답변 초안 워크플로 분석

## 라이브러리 메타데이터
- 사례 ID: impact-agent-20260605-neighborhood-repair-router
- 분야: 지역
- 업무 유형: 분류
- 에이전트 역할: 라우팅
- 데이터 특성: 주민 제보 텍스트와 사진 설명이 섞인 반정형 데이터
- 정형/비정형 여부: 반정형
- 개인정보 포함 여부: 공개 데이터 중심. 연락처와 상세 주소가 있으면 제거 필요
- 연결 수준: 수동 업로드
- MCP/CLI 연결 여부: MCP/CLI 연결 없음. 제보 텍스트 수동 업로드
- 사람 검토: 최종 검토
- 재사용 가능성: 그대로 재사용 가능
- 산출물 형식: md

## 기존 업무 흐름
- 업무 시작 조건: 주민 제보 접수
- 데이터 수: 데모 5건, 실제 업무 주 10-25건
- 반복 빈도: 수시 접수, 비나 행사 이후 증가
- 기존 소요 시간: 건당 3-7분
- 입력 데이터 출처: 카카오톡과 구글폼
- 기존 단계: 활동가가 제보를 읽고 분류한 뒤 답변을 직접 작성한다.
- 기존 산출물: 스프레드시트 분류표와 답변 메시지
- 불편한 지점: 채널이 흩어져 누락이 생기고 답변 톤이 달라진다.

## 자동화 대상
- 선정한 업무: 제보 5건 분류표, 담당 후보, 위치 확인 질문, 답변 초안 작성
- 선정 이유: 텍스트 제보 5건 묶음으로 활동가의 실제 처리 흐름을 끝까지 보여줄 수 있다.
- 제외한 업무: 지도 연동과 기관 자동 접수

## AI 에이전트 설계
- 에이전트 입력 조건: 개인정보를 제거한 제보 텍스트 5건
- 에이전트 산출물 조건: 제보 분류표, 담당 후보, 위치 확인 질문, 주민 답변 초안
- 판단 경계: AI는 담당 후보를 제안하고 활동가가 최종 확인한다.
- 사용 도구: Codex, ChatGPT
- 바뀐 업무 흐름: 제보 텍스트 입력 -> AI 분류/답변 초안 -> 활동가 검토 -> 주민 답변

## 사람 검토와 개입
- 주요 검수 포인트: 담당 후보가 맞는지, 사유지/공공영역 판단이 안전한지, 주민 답변 톤이 적절한지 확인
- 실패 시 사람이 이어받는 방식: 담당 후보가 불확실하면 기존 담당자 문의 방식으로 전환한다.
- 예상 오류 유형: 담당 기관 오분류, 사유지/공공영역 혼동
- 평가 샘플: 익명화 제보 5건과 기대 유형

## 데이터와 접근
- 개인정보 처리: 연락처와 상세 주소를 제거한다.
- 권한/접근 이슈: 해당 없음
- 결과물의 공유 가능 범위: 공개 가능

## 영향과 재사용
- 기존 대비 소요 시간 변화: 건당 3-7분에서 초안 생성 1분 이내
- 품질 변화: 답변 형식과 분류 기준 일관성 향상
- 재사용 방법: 지역별 담당 후보 목록과 답변 톤을 바꾼다.
- 유지관리 주체: 마을 활동가
- 다음 개선: 사진 설명과 위치 정보를 함께 받기
