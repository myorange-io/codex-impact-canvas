## 기록

### 2026-06-05 문제 정의
- **정한 것 / 한 것**: 필수 마크다운 산출물 구조와 3시간 MVP 범위를 정리했다.
- **왜**: 팀별 결과물을 같은 구조로 남겨 이후 사례집과 워크플로 라이브러리로 재사용하기 위해서다.
- **어떻게**: 반응형 질문 답변을 표준 필드로 정규화해 PLAN.md, WORKFLOW_ANALYSIS.md, CASE_STUDY.md, MEMORY.md에 기록했다.
- **막힌 점 / 바꾼 점**: 없음
- **배운 것 / 다음**: 샘플 데이터로 MVP를 구현하고 검수 포인트를 보완한다.

### 2026-06-06 MVP 범위 보강
- **정한 것 / 한 것**: 제보 1건 답변 초안 중심 MVP를 제보 5건 분류표, 담당 후보, 위치 확인 질문, 주민 답변 초안까지 포함하는 완결형 흐름으로 보강했다.
- **왜**: 활동가가 실제로 느끼는 문제는 한 건 답변보다 여러 채널 제보를 누락 없이 처리 순서로 정리하는 것이기 때문이다.
- **어떻게**: 샘플 입력을 5건 묶음으로 바꾸고, 분류에서 담당 후보, 위치 확인, 답변 초안까지 이어지는 업무 단계를 정리했다.
- **막힌 점 / 바꾼 점**: 사진 판독, 지도 연동, 기관 자동 접수는 3시간 안에서는 제외했다.
- **배운 것 / 다음**: 지역별 담당 후보 목록과 우선순위 표시 기준을 샘플로 더 검수한다.
