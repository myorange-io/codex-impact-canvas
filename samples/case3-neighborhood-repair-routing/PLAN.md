# 생활 불편 제보 분류와 답변 초안 실행 계획

## 팀
- 사회혁신가: 정지역
- 개발자: 한개발

## 문제
- 실제 업무 문제: 주민 제보가 자유형식으로 들어와 담당 분야 분류와 답변 초안 작성이 늦어진다.
- 가장 어려움을 겪는 사람: 마을 활동가와 생활 불편을 제보한 주민
- 문제가 발생하는 순간: 비가 온 뒤 시설 파손 제보가 여러 채널로 동시에 들어올 때

## 현재 업무 흐름
- 입력 자료: 카카오톡/폼으로 들어온 제보 텍스트와 사진 설명
- 처리 순서: 제보 확인 -> 유형 분류 -> 담당자 추정 -> 답변 작성 -> 공유
- 판단 기준: 안전 위험, 청소, 시설 파손, 사유지 여부, 담당 기관
- 현재 산출물: 분류표와 주민 답변 초안
- 현재 사용 도구: 카카오톡, 구글폼, 스프레드시트

## 기대 변화
- 기대 변화: 제보 분류와 1차 답변 초안을 빠르게 만들어 대응 누락을 줄인다.
- 관찰 가능한 개선: 샘플 제보 1건에서 유형, 담당 후보, 답변 초안이 나온다.

## 3시간 MVP
- 핵심 기능 1개: 주민 제보 텍스트를 넣으면 유형과 담당 후보, 답변 초안을 생성한다.
- 데모 입력: 익명화한 제보 텍스트 1건
- 데모 산출물: 제보 유형, 담당 후보, 주민 답변 초안
- 성공 기준: 샘플 제보가 올바른 유형으로 분류되고 활동가가 수정 가능한 답변 초안이 나온다.

## 제외 범위
- 오늘 하지 않을 것: 사진 판독, 담당 기관 자동 접수
- 다음 단계 아이디어: 사진 업로드와 지도 위치 연결

## 3시간 구현 계획
- 0-30분: 제보 유형과 답변 톤을 정한다.
- 30-90분: 분류/답변 프롬프트를 만든다.
- 90-150분: 텍스트 입력 데모를 만든다.
- 150-180분: 샘플 제보 5건으로 분류 정확성을 확인한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 제보 5건 일괄 분류표 추가
- 후보 1 시작 조건: 텍스트 제보 1건 분류와 답변 초안이 통과하고 40분 이상 남았을 때
- 후보 1 구현 내용: 여러 제보를 줄 단위로 입력받아 유형, 담당 후보, 답변 초안을 표로 출력한다.
- 후보 1 데모 산출물: 제보 5건의 유형별 분류표와 답변 초안
- 후보 1 예상 추가 시간: 40분
- 후보 1 검수 기준: 각 제보가 하나의 유형으로 분류되고 담당 후보가 빠짐없이 나온다.
- 후보 2 작업: 위치 정보 확인 질문 추가
- 후보 2 시작 조건: 핵심 기능 결과가 안정되고 25분 이상 남았을 때
- 후보 2 구현 내용: 주소나 위치가 불명확한 제보에 대해 주민에게 되물을 확인 질문을 자동으로 붙인다.
- 후보 2 데모 산출물: 위치 확인 질문이 포함된 주민 답변 초안
- 후보 2 예상 추가 시간: 25분
- 후보 2 검수 기준: 위치가 부족한 샘플에서 추가 질문이 생성되고, 위치가 충분한 샘플에는 불필요한 질문이 붙지 않는다.
