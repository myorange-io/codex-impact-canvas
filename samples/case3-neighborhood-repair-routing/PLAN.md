# 생활 불편 제보 분류와 답변 초안 실행 계획

## 팀
- 사회혁신가: 정지역
- 개발자: 한개발

## 문제
- 실제 업무 문제: 주민 제보가 자유형식으로 들어와 담당 분야 분류와 답변 초안 작성이 늦어진다.
- 가장 어려움을 겪는 사람: 마을 활동가와 생활 불편을 제보한 주민
- 문제가 발생하는 순간: 비가 온 뒤 시설 파손 제보가 여러 채널로 동시에 들어올 때

## 현재 업무 흐름
- 입력 자료: 카카오톡/폼으로 들어온 제보 텍스트와 사진 설명
- 처리 순서: 제보 확인 -> 유형 분류 -> 담당자 추정 -> 답변 작성 -> 공유
- 판단 기준: 안전 위험, 청소, 시설 파손, 사유지 여부, 담당 기관
- 현재 산출물: 분류표와 주민 답변 초안
- 현재 사용 도구: 카카오톡, 구글폼, 스프레드시트

## 기대 변화
- 기대 변화: 제보 분류와 1차 답변 초안을 빠르게 만들어 대응 누락을 줄인다.
- 관찰 가능한 개선: 샘플 제보 5건에서 유형, 담당 후보, 위치 확인 질문, 답변 초안이 표로 나온다.

## 3시간 MVP
- 해결할 좁은 문제: 여러 채널에서 들어온 생활 불편 제보 5건을 활동가가 오늘 처리할 순서와 답변 초안으로 정리한다.
- 핵심 기능 1개: 주민 제보 텍스트 5건을 넣으면 유형, 담당 후보, 위치 확인 질문, 주민 답변 초안을 생성한다.
- 포함할 업무 단계: 제보 요약 -> 유형 분류 -> 담당 후보 제안 -> 위치 부족 여부 확인 -> 주민 답변 초안 작성
- 데모 입력: 익명화한 제보 텍스트 5건 묶음
- 데모 산출물: 제보 분류표, 담당 후보, 위치 확인 질문, 주민 답변 초안
- 성공 기준: 각 제보가 하나의 유형으로 분류되고 담당 후보와 주민에게 보낼 다음 답변이 빠짐없이 나온다.
- 3시간 안/밖 판단: 안 - 텍스트 묶음과 수동 검토만 필요하며 지도 연동과 기관 접수는 제외한다.

## 제외 범위
- 오늘 하지 않을 것: 사진 판독, 담당 기관 자동 접수
- 다음 단계 아이디어: 사진 업로드와 지도 위치 연결

## 3시간 구현 계획
- 0-30분: 제보 유형과 답변 톤을 정한다.
- 30-90분: 분류/답변 프롬프트를 만든다.
- 90-150분: 텍스트 입력 데모를 만든다.
- 150-180분: 샘플 제보 5건으로 분류 정확성을 확인한다.

## 빠르게 끝났을 때 이어서 할 작업 후보
- 후보 1 작업: 처리 우선순위 색상 표시 추가
- 후보 1 시작 조건: 제보 5건 분류표와 답변 초안이 통과하고 35분 이상 남았을 때
- 후보 1 구현 내용: 안전 위험, 반복 민원, 위치 불명확 여부를 기준으로 우선순위 표시를 붙인다.
- 후보 1 데모 산출물: 우선순위 표시가 포함된 제보 처리표
- 후보 1 예상 추가 시간: 40분
- 후보 1 검수 기준: 활동가가 먼저 처리할 제보를 표에서 바로 구분할 수 있다.
- 후보 2 작업: 운영 기록용 요약 추가
- 후보 2 시작 조건: 답변 초안이 안정되고 25분 이상 남았을 때
- 후보 2 구현 내용: 각 제보의 처리 상태, 담당 후보, 다음 행동을 운영 기록용 한 줄 요약으로 만든다.
- 후보 2 데모 산출물: 스프레드시트에 붙여 넣을 운영 기록 요약
- 후보 2 예상 추가 시간: 25분
- 후보 2 검수 기준: 활동가가 답변 후 남길 기록을 다시 작성하지 않아도 된다.
