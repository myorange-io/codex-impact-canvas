# 만들기 기록 - 후원 미납 위험 목록 생성

> 이 파일은 이 MVP를 어떤 과정으로 만들었는지 남기는 기록입니다.
> 나중에 복기하거나 사회혁신 AI 에이전트 워크플로 사례로 정리할 때 씁니다.

## 한눈에 보기
- 무엇을: 후원자 관리 과정에서 후원 내역 CSV를 사람이 확인하며 미납 또는 중단 위험 대상을 찾기 어렵다.
- 누구를 위해: 후원자 관리를 담당하는 실무자와 지속적인 관계 관리가 필요한 후원자
- 핵심 흐름: 익명화 CSV 업로드 -> AI/스크립트 미납 판별 -> 위험 목록 생성 -> 담당자 최종 검토 -> 연락 여부 결정
- 스택/도구: Codex, CSV 처리 스크립트 또는 스프레드시트
- 시작: 2026-06-10

## 기록

### 2026-06-10 MVP 범위 합의
- **정한 것 / 한 것**: 후원자 관리 문제를 후원 내역 CSV 업로드 후 최근 2개월 미납/중단 위험 목록을 자동 생성하는 3시간 MVP로 확정했다.
- **왜**: 처음에는 후원자 관리 전체를 줄이고 싶다는 요구가 있었지만, 3시간 안에는 최근 2개월 미납자 찾기라는 반복 판단 하나를 끝까지 자동화하는 편이 검수 가능하기 때문이다.
- **어떻게**: 입력은 익명화한 후원 내역 CSV 1개, 출력은 최근 2개월 미납/중단 위험 목록으로 고정하고, 사람은 실제 미납 여부와 예외 사유를 최종 확인하도록 나누었다.
- **막힌 점 / 바꾼 점**: 팀이 AI와 사람의 역할 분담을 바로 정하기 어려워 기본 분담안을 제안했다. 실제 연락/발송 자동화와 외부 시스템 연동은 제외 범위로 뺐다.
- **배운 것 / 다음**: 샘플 CSV를 준비해 정상 납부자 오탐과 미납자 누락 여부를 검수한다. 시간이 남으면 연락 우선순위 표시와 안내 문자 초안 생성을 이어서 구현한다.
